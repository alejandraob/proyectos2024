# Instrucciones para Claude Code

## Proyecto: MiTurno
Sistema de turnos SaaS multi-tenant (Laravel 10 + Vue.js 3 + MariaDB)

## Reglas de Documentación

**IMPORTANTE:** Después de cada cambio significativo (fix, feature, mejora), actualizar la documentación:

1. **docs/FIXES.md** - Documentar fixes y features con:
   - Número secuencial (Fix #XXX o Feature #XXX)
   - Fecha
   - Problema/Descripción
   - Causa (si es fix)
   - Solución
   - Archivos modificados

2. **docs/TODOLIST_FRONTEND.md** - Actualizar:
   - Marcar items completados [x]
   - Agregar nuevos items si corresponde
   - Actualizar estructura de archivos si se crearon nuevos

3. **docs/TODOLIST_BACKEND.md** - Actualizar:
   - Marcar items completados [x]
   - Agregar nuevos items si corresponde
   - Documentar nuevas migraciones, modelos, controllers, rutas

4. **docs/PROYECTO_DIARIO.md** - Actualizar:
   - Trabajo realizado en la sesión actual
   - Problemas encontrados
   - Tabla de progreso del MVP

## Convenciones del Proyecto

- Idioma: Español (Argentina) para UI y documentación
- Comentarios en código: Español
- Variables y funciones: Inglés o Español según contexto
- Commits: Español

## Archivos de Documentación

| Archivo | Propósito |
|---------|-----------|
| docs/FIXES.md | Historial de errores y soluciones |
| docs/TODOLIST_FRONTEND.md | Tareas del frontend |
| docs/TODOLIST_BACKEND.md | Tareas del backend |
| docs/PROYECTO_DIARIO.md | Registro diario y progreso |
| docs/PRUEBAS.md | Registro de pruebas API |
